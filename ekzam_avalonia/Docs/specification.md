# Спецификация проекта

## 1. Общая информация
- **Название:** Лавка «Матье» - управление услугами
- **Тип:** desktop-приложение на Avalonia
- **Парадигма:** code-behind, без MVVM
- **База данных:** PostgreSQL
- **Провайдер БД в коде:** Npgsql (синхронные запросы)

## 2. Технический стек
- .NET 8
- Avalonia 11
- Avalonia ListBox (карточки услуг)
- PostgreSQL 14+ (рекомендуется 15/16)
- xUnit для unit-теста

## 3. Развертывание
- **Клиентская машина (разработчик/оператор):** Windows 10/11, Visual Studio 2022 или Rider.
- **Сервер БД:** PostgreSQL локально или на отдельном сервере.
- **Connection string:** `Services/DbSettings.cs`.

## 4. Реализованные функции
1. Фиксированное окно, центр экрана, кастомные кнопки закрытия/сворачивания.
2. Диалог подтверждения при закрытии.
3. Загрузка услуг из таблицы `services` в карточный список.
4. Пагинация по 3 записи.
5. Сортировка по названию.
6. Добавление услуги через отдельное окно.
7. Редактирование услуги через отдельное окно.
8. Обновление `last_modified_date` при add/update.
9. Вывод изображений товаров в таблице.
10. Иконка приложения из предоставленного логотипа.

## 5. Структура проекта
- `Models` - простые модели данных.
- `Services` - репозиторий на Npgsql и бизнес-логика.
- `Views` - окна Avalonia (`MainWindow`, `ServiceEditWindow`).
- `Assets/Images` - логотип и изображения товаров.
- `Sql` - скрипты создания схемы и тестовых данных.
- `Docs` - документация и тест-кейсы.
- `ekzam_avalonia.Tests` - unit-тесты.

## 6. Результаты тестирования
- Unit-тест `UpdateService_ChangesLastModifiedDate` подтверждает, что метод обновления услуги меняет `last_modified_date`.
- Ручные тест-кейсы описаны в `Docs/test-cases.md` (5 кейсов).

## 7. Ограничения
- Версия без MVVM и без асинхронности по заданию.
- Валидация формы базовая.
- Для итогового отчета DOCX/PDF формируется из markdown-документов папки `Docs`.
